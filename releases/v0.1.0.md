## Release summary

AgentPay SDK `v0.1.0` is the first local-first release of the AgentPay SDK for self-custodial, policy-aware wallet operations on macOS.

This release includes the following high-level capabilities:

1. A macOS local runtime for self-custodial wallet setup, signing, and transaction execution.
2. One-click installation via `curl -fsSL https://wlfi.sh | bash` and a source-based installation path for engineering teams.
3. First-run wallet setup, wallet reuse, encrypted offline backup export, backup verification, and wallet restore.
4. Local daemon-managed signing with policy enforcement before signatures are produced.
5. Token-first policy management through the terminal UI, including limits, overrides, and manual approval overlays.
6. Native transfers, ERC-20 transfers, ERC-20 approvals, raw broadcast flows, and local manual approval handling.
7. Support for EVM-compatible chains, including common defaults such as Ethereum and BSC, with USD1 as the default asset path.
8. A plugin system for third-party integrations and open-source contribution, with Bitrefill included as an example integration.
9. A foundation for the next release, which is planned to expand into relay, gasless transaction support, and EIP-3009 flows.

## Supported environments

AgentPay SDK `v0.1.0` supports the following runtime environment:

- macOS as the primary supported runtime platform
- Node.js `20+` for the CLI launcher
- one-click installation using prebuilt macOS runtime bundles
- source installation from the repository using the documented `pnpm` and `npm` workflow

This release is designed for local-first operation and is intended for operators who want the wallet, approval path, and signing boundary to remain on the local machine.

## Installation and upgrade paths

This release supports the following installation and upgrade paths:

- one-click install via `curl -fsSL https://wlfi.sh | bash`
- one-click skill-only install via `curl -fsSL https://wlfi.sh | bash -s -- --skills-only` for agent integrations and adapters
- source install from a repository checkout
- local runtime refresh by rerunning `curl -fsSL https://wlfi.sh | bash`
- source-based upgrade by pulling the latest source and reinstalling the launcher and Rust binaries
- wallet reattachment after runtime refresh with `agentpay admin setup --reuse-existing-wallet`

The one-click installer can also install agent-facing skill packs and workspace adapters during setup.

## Wallet and key management

AgentPay SDK `v0.1.0` includes the following wallet and credential lifecycle features:

- first-run wallet creation with `agentpay admin setup`
- reuse of an existing local wallet with `agentpay admin setup --reuse-existing-wallet`
- restore of the same wallet from an encrypted offline backup with `agentpay admin setup --restore-wallet-from <PATH>`
- encrypted offline wallet backup export
- backup verification before operational use
- managed daemon installation and recovery as part of setup
- local agent credential provisioning during setup
- local secret storage using macOS Keychain for the agent auth token and daemon password
- encrypted daemon state persistence for local operational state

This release is intended to give operators a complete local wallet lifecycle from initial setup through recovery.

## Policy and approval workflows

AgentPay SDK `v0.1.0` includes the following policy and approval features:

- local policy evaluation before signing
- per-token default limits
- destination-specific overrides
- manual approval overlays
- token and network inventory editing through the terminal UI
- shared-config editing through direct admin CLI commands
- local listing of manual approval requests
- local approval and rejection of pending manual approval requests
- resume support for approved broadcast-backed requests
- automatic waiting behavior for `--broadcast` flows that pause for approval

The terminal UI is the primary operator surface for policy authoring and policy refresh in this release.

## Supported chains and default asset

AgentPay SDK `v0.1.0` is focused on EVM-compatible chains.

This includes:

- built-in support for common EVM chain profiles such as Ethereum and BSC
- support for additional EVM-compatible chain profiles through the admin CLI and terminal UI
- native asset handling on supported configured chains
- USD1 as the default asset path for the standard payment workflow
- built-in USD1 profiles on Ethereum and BSC
- ERC-20 token metadata fetch and token inventory management through the terminal UI

This release is intended to make the default operator path straightforward: use an EVM-compatible chain, with USD1 as the default value asset and the native asset as gas.

## Transaction and plugin capabilities

AgentPay SDK `v0.1.0` supports the following core transaction flows:

- native asset transfers with `agentpay transfer-native`
- ERC-20 transfers with `agentpay transfer`
- ERC-20 allowance approvals with `agentpay approve`
- policy-checked raw broadcast request handling with `agentpay broadcast`
- sign-and-send transaction flows for supported EVM requests
- nonce reservation and replay-protection handling in the daemon signing path
- JSON output modes for automation and tooling integration

This release also includes a plugin system for third-party integrations and contributions.

The plugin system is intended to let developers extend AgentPay SDK with additional payment and workflow integrations while reusing the same wallet, signing, and policy path.

Bitrefill is included as an example integration in this release. It demonstrates how a third-party integration can plug into the existing transaction, signing, and policy flow without introducing a separate wallet model.

## Planned for the next release

The next release is planned to expand AgentPay SDK in the following areas:

### Relay

- relay-backed approval coordination
- browser-oriented approval transport for teams that want an approval console workflow
- broader approval-routing options while preserving local signing control

### Gasless

- gasless transaction support for selected flows
- improved user experience for wallets that should transact without direct gas-asset management
- better sponsored-transaction and delegated execution paths

### EIP-3009

- native EIP-3009 flow support
- authorization-based transfer workflows
- expanded token UX for assets that support transfer-by-authorization patterns

AgentPay SDK `v0.1.0` establishes the local runtime foundation. The next release is planned to build on that base with broader approval transport and richer transaction primitives.
